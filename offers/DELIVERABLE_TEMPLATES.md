# Deliverable Templates

## Template A — AI Stack Rescue (2h)
Client receives:
1. Incident summary (symptom, likely cause)
2. Actions performed (exact commands or equivalent)
3. Changed files/units list
4. Verification block:
   - service status
   - route checks
   - feature check (e.g. TTS synth)
5. Rollback steps
6. Next-risk notes

## Template B — Local AI Setup + TTS (1 day)
Client receives:
1. Architecture snapshot (components + ports)
2. Config baseline (sanitized)
3. Verification script and output
4. Drift guard/monitoring status
5. Operations notes (restart/recover/validate)
6. Scope boundaries for further work

## Template C — Ops Audit + Launch Gate
Client receives:
1. Current posture summary (PASS/WARN/FAIL)
2. Failed-unit triage (core vs non-core)
3. Repair/disable decisions with rationale
4. Launch gate result
5. Evidence pack index
6. Re-enable plan for deferred components
